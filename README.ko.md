# crier

**코딩 에이전트가 무슨 일이 있었는지 말해준다 — 한 문장으로, 필요할 때만.**

한국어 · 영어 · 일본어. 전부 로컬에서 돈다. API 키도, 클라우드도, 글자당 과금도 없다.

[English README](README.md)

```
🔔  "메모리 누수 고쳤고 테스트 세 개 다 통과했어요. 커밋할까요?"
🔔  "rm 실행해도 될까요?"
🔔  "요청 한도에 걸렸습니다."
```

알림음이 아니다. **화면을 안 보고도 행동할 수 있는 문장**이다.

---

## 왜

에이전트가 일하는 동안 딴 창으로 넘어간다. 딩 소리가 난다. 돌아온다 — 그런데 결국 **화면을 읽어야** 뭐가 끝났는지 안다.

[peon-ping](https://github.com/PeonPing/peon-ping)(4.9k★)이 트리거 모델은 옳다는 걸 증명했다. 에이전트 생명주기에 훅을 걸고 소리를 낸다. 하지만 고정된 소리는 *뭔가* 일어났다는 것까지만 알려준다. **뭐가** 일어났는지는 못 알려준다. 그래서 결국 화면을 본다.

crier는 그 소리 자리에 **문장**을 넣는다.

무엇을 말할지 정하는 기준은 질문 하나다. **이걸 듣고 내 행동이 바뀌는가?**

| 이벤트 | 말하는 것 | 행동이 바뀌나 |
|---|---|---|
| 작업 완료 | *"메모리 누수 고쳤고 테스트 세 개 다 통과했어요. 커밋할까요?"* | ✅ 돌아와서 확인 |
| 허가 필요 | *"rm 실행해도 될까요?"* | ✅ 승인하거나 말거나 |
| 요청 실패 | *"요청 한도에 걸렸습니다."* | ✅ 기다리거나 딴 일 |
| 오래 대기 | *"기다리고 있어요."* | ✅ 돌아옴 |
| ~~에이전트 시작~~ | ~~*"작업을 시작합니다"*~~ | ❌ **기본 off** |

마지막 줄이 이 설계의 축소판이다. 방금 엔터 친 사람한테 "작업 시작합니다"라고 말해봐야 아무것도 안 바뀐다. **문구를 아무리 다듬어도 안 바뀐다 — 문장이 나쁜 게 아니라 이벤트에 할 말이 없는 것이니까.** 그래서 껐다.

## 문장은 누가 쓰는가

에이전트가 쓴다.

응답 끝에 낭독용 한 줄을 붙이게 한다.

```
[말] 메모리 누수 고쳤고 테스트 세 개 다 통과했어요. 커밋할까요?
```

**추가 API 호출도, 추가 키도 없다.** 방금 그 작업을 한 모델 — 그 턴이 무슨 일이었는지 아는 유일한 존재 — 이 한 줄을 더 쓸 뿐이다. 출력 토큰 몇 개. **코딩 에이전트를 쓰고 있다면 crier에 필요한 건 이미 다 갖고 있다.**

```bash
bin/voiced prompt install     # 에이전트 메모리 파일에 지시문을 넣는다
```

안 넣어도 된다. 그러면 규칙으로 뽑는다 — 마지막 문장이 질문이면 그것(당신이 답해야 할 것), 아니면 첫 문장(결론이 나오는 자리). 잘 굴러가지만, **에이전트가 쓰는 문장이 어떤 규칙보다 낫다.**

내용이 없는 안내(허가·에러·대기)는 [`locales.json`](locales.json)의 캔드 문장이다. 모델이 보탤 게 없으니까 — 할 말이 이미 정해져 있다.

## 설치

```bash
git clone https://github.com/pg-Parunson/crier
cd crier
./install.sh          # 또는: ./install.sh codex | gemini | cursor
```

그리고 에이전트 세션을 다시 시작한다.

[uv](https://docs.astral.sh/uv/)와 macOS가 필요하다. (리눅스는 `aplay` 필요 — 재생 미검증. PR 환영.)

## 지원 에이전트

| 에이전트 | 턴 종료 + 본문 | 허가 + 무슨 명령인지 |
|---|---|---|
| **Claude Code** | ✅ | ✅ |
| **Codex CLI** | ✅ | ✅ |
| **Gemini CLI** | ✅ | ✅ |
| **Cursor** | ✅ | ✅ |

Claude Code와 Codex는 이벤트 이름도 페이로드도 **똑같아서** 훅을 공유한다. Gemini와 Cursor는 같은 말을 다른 단어로 하는데 [`announce.py`](bin/announce.py)가 변환한다. 에이전트 추가는 [`bin/wire.py`](bin/wire.py)의 라우팅 테이블 한 줄이지, 통합 작업이 아니다.

내용 없는 핑만 주는 에이전트(Aider)는 브리핑을 못 싣는다. 거기선 crier가 틀린 도구다 — peon-ping을 써라.

## 목소리

[Supertonic](https://github.com/supertone-inc/supertonic) — 로컬 ONNX 모델. GPU도 네트워크도 필요 없다. 목소리 10종(F1–F5, M1–M5), 31개 언어.

```bash
bin/voiced voices            # 10개 다 듣기
bin/voiced voice M3
bin/voiced lang ja           # ko | en | ja
bin/voiced tone playful      # plain | friendly | playful
bin/voiced name "민수님"      # 매번 말고 가끔만 부른다
bin/voiced demo              # 전 이벤트 듣기
```

**한국어가 진짜 된다.** 이걸 굳이 적는 이유는, 대부분의 오픈 TTS가 **조용히 안 되기** 때문이다. Kokoro-82M은 한국어가 아예 없다(메인테이너가 학습에서 뺐다). Chatterbox Multilingual은 한국어 CER 70.9% — Resemble AI가 직접 공표한 숫자다. Piper는 `ko_KR` 보이스가 없다. XTTS-v2는 한국어가 되지만 비상업 라이선스인데 회사가 2024년에 폐업해서 **영원히 해제 불가**다. Supertonic은 한국어 문장 속 `useEffect`를 *"useEffect"*로 읽는다 — 알아들을 수 없는 한글 음차로 뭉개지 않고.

M-시리즈 맥 실측 RTF **0.159** — 5초짜리 문장 합성에 약 0.8초. (Supertonic 마케팅은 0.012라고 하는데 **재현되지 않았다.**) 그래도 충분히 빠른 이유는 아래 효과음 항목에 있다.

## 말투

| | 허가 | 대기 |
|---|---|---|
| `plain` | *"rm 실행해도 될까요?"* | *"대기 중입니다."* |
| `friendly` | *"rm 좀 돌려볼게요. 허락해 주실래요?"* | *"기다리고 있어요."* |
| `playful` | *"rm 명령, 사인 한 번만 주시죠."* | *"심심합니다. 시키실 거 없나요?"* |

말투는 캔드 문장에도, **에이전트에게 주는 지시문에도** 들어간다. 그래서 에이전트가 직접 쓰는 문장까지 성격이 이어진다. 여기서 페르소나가 공짜인 이유다 — 모델이 아니라 프롬프트 한 줄이니까.

## 효과음

목소리가 예고 없이 훅 들어오면 놀란다. 그래서 종을 먼저 친다.

효과음은 **설치할 때 합성한다**([`bin/earcons.py`](bin/earcons.py)). 파일로 번들하지 않는다 — 라이선스 걸릴 것도, 출처 표기할 것도 없다. 감쇠 포락선 아래의 배음 몇 개, 종을 때린 소리다. **올라가면** 묻거나 알리고, **내려가면** 마무리하거나 경고한다. 전부 0.5초 미만.

값을 두 번 한다: **종이 울리는 동안 뒤에서 음성을 합성한다.** 종이 잦아드는 순간 말이 시작되니, 모델이 생각하는 소리를 들을 일이 없다.

## 소음 제어

계속 깔아둘지 말지가 여기서 갈린다.

병렬 팬아웃은 훅을 **에이전트 수만큼** 터뜨린다. 프로젝트 10개를 동시에 굴리면 순진한 구현은 소음 공격이 된다.

1. **턴당 한 번.** 서브에이전트 안내는 그 턴(`prompt_id`)의 **첫 에이전트에서만** 울린다. 8개든 30개든 한 번, 그리고 다음 프롬프트까지 조용하다. (쿨다운만으론 못 한다 — 무거운 턴은 몇 분에 걸쳐 계속 새 에이전트를 띄우니 초반 버스트만 눌릴 뿐이다.)
2. **쿨다운** (2.5초, 세션 간 공유) — 프로젝트 10개가 서로 겹쳐 떠들지 않는다.
3. **우선순위.** 허가와 에러는 **쿨다운을 뚫는다.** 수다스러운 안내가 정작 *"rm 실행해도 될까요?"* 를 잡아먹으면 안 된다.

그래도 많으면 `config.json`에 이벤트별 스위치가 다 있다.

## 끼어들기

타이핑을 시작하면 말하던 게 단어 중간에서 멈춘다.

## 왜 훅인가 (MCP 아님)

- **훅은 모델의 협조가 필요 없다.** MCP 툴은 모델이 부르기로 *결심*해야 실행된다 — "작업 끝"이라고 판단하는 순간 음성 루프가 조용히 죽는다.
- 훅은 에이전트 설정 파일에 있고, 그 파일을 **터미널과 IDE 확장이 공유**한다. VS Code의 Claude 확장은 MCP 서버를 발견조차 못 한다.
- 훅은 ~50ms에 리턴하고 재생은 분리된 자식에게 넘긴다. 턴이 발화가 끝나길 기다리지 않는다.
- stdout에 아무것도 안 쓴다 → **에이전트의 제어 흐름에 영향을 줄 수 없다.** crier가 터져도 세션은 안 터진다.

## 설정

`config.json` — 이벤트마다 다시 읽으므로 **즉시 반영**된다.

| | |
|---|---|
| `lang` `tone` `voice` `call_name` `name_chance` | 누구인가 |
| `events.*` | 무엇을 말하는가 |
| `cooldown_sec` `max_chars` | 얼마나 |
| `earcons.enabled` `gap_ms` | 종 |
| `marker` | 에이전트가 낭독용 줄을 쓸 때 붙이는 접두어 |
